CSS Grid Debugger

A tiny little console utility to make building grid-based designs easier

Download the file (5kb) or install the CSS Grid Debugger bookmarklet.

Designing with a grid system is great. Implementing the details can be hard

CSS Grid—or frameworks like Foundation and Bootstrap—make building layouts with a well-defined grid system easy. Even with these powerful tools, it's sometimes it's a bit difficult to translate complex grid-based designs to a functional HTML version. It's easy for the details to get missed, espectially when dealing with responsive designs that need to work across multiple screen sizes.

This script adds some overlays to help make sure your designs are lining up to a vertical and horizontal grid.

  • highlighting horizontal baseline grids
  • highlighting column grids

Adding the script

Include the grid_debug.js script in your document. You probably need to override the defaults to match your design's grid system. You can do that with the GRID.setBreakpoint function immediately after your script include.

<script src="/path/to/grid_debug.js"></script>

You should include the script at the bottom of the BODY section of the document.

Setting default values

There are a few configuration options you can set:

  • width - the width of the container box (default value is 100%)
  • column-count - the number of columns your grid system uses (default value == 12)
  • column-gutter - the width of the space between your columns
  • column-margin - the width of the space on the sides of the columns relative to the container box
  • column-color - the color of the column overlay
  • baseline-height - A CSS-valid height for the grid. Remember to include px, %, or em units! (default value == '9px')
  • baseline-color - the color of the baseline overlay
  • baseline-color-inverted - the color of the baseline overlay when in inverted-mode

Try it yourself

Fire up the Javascript Console and type:

GRID.baseline()

Now, press SHIFT a couple of times.

GRID.columns()

Now, hit ESC.

Adjusting the values to match your grid system

Updating the defaults

  <script src="grid_debug.js"></script>
  <script>
  GRID.setBreakpoint('desktop', {
    'width': '1170px',
    'column-count': 12,
    'column-gutter': '2.5%',
    'column-margin': '0px',
    'baseline-height': '9px'
  });
  </script>

Adding new breakpoints

  <script src="grid_debug.js"></script>
  <script>
    GRID.setBreakpoint('mobile', {
      'column-count': 4,
      'column-gutter': '20px',
      'column-margin': '20px',
      'baseline-height': '9px'
    });

    // specify when to use the breakpoint 
    // you defined with the `use` function
    if (window.innerWidth < 999) {
      GRID.use('mobile');
    }
  </script>

You can set up as many breakpoints as you need.

Usage

As you are working on your design, you can use the following commands from the javascript console:

GRID.baseline() - overlay the baseline grid. You may optionally pass in a CSS-valid height for the grid (remember to include px, %, or em units). I

GRID.columns() - overlay the columns using your chosen grid framework

GRID.clear() - clear any overlays

Keyboard shortcuts

When the grids are visible, you can press ESC to clear them. For the baseline grids, the following keyboard commands are available:

  • j key - jog the grid up by one pixel at a time
  • k key - jog the grid down by one pixel at a time
  • SHIFT key - invert the colors on the grid. Useful on dark backgrounds.
